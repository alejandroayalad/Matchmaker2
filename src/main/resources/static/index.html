Make The Extraction Look Useful.

The app now exposes richer import previews and surfaces the fields that were already being captured but not shown.

Stored Jobs, Ordered And Expanded.

This view prioritizes recent active jobs and shows description, requirements, salary, recruiter, modality, timestamps, and any other non-empty fields. The goal is to make extraction quality obvious instead of hiding everything behind a title.

What changed

`/api/jobs` is now sorted by active status and recency. The frontend also distinguishes real data from placeholders, so the cards emphasize useful extracted content rather than empty defaults.

Matched Jobs With Context.

The matched view now uses the same rich card layout, so you can see why a score is high or low without flipping back to the jobs feed.

Tip

If a strong-looking job scores poorly, the missing field is often `jobType`, `experienceLevel`, or the description text. Those are visible here now.

Scrape And Inspect.

Run the scrapers and immediately inspect what came back, including the description excerpt and inferred job type when available.

Scope

This page shows transient scraper results; the Jobs page shows what is already stored in the app.

URL Import Preview.

Use this to test page-level importers and confirm that description, salary, modality, and seniority are being inferred correctly.

Why this matters

The imported URL flow now infers job type the same way the email and scraper flows do, which makes the matched page more trustworthy.

Email Import, Now With Proof.

The response includes counters plus two preview streams: what the parser extracted from the email, and which jobs were actually saved. This makes it obvious whether the gap is parsing, deduplication, or persistence.

New payload

`/api/import/email` now returns `extractedJobs` and `savedJobs` in addition to the counts. LinkedIn email parsing also captures summary and insight chips when the email markup contains them.
